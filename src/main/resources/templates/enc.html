Keys To Try

Cipher Description Example Keys
Caesar Cipher The binary key is converted to a number which represents the 'shift' e.g. 1 is converted to 1, 10 is converted to 2. You are limited to a max of 4, so 100 is max [1], [10], [11], [100]
Block Cipher The message has to be evenly divisible by the key, e.g. if the message was 8 bits, the key would need to be 2, 4, or 8 bits so it leaves no remainder when divided by. Remember, each character in the input message is 1 byte and so 8 bits. Therefore 'Text' has 4 bytes or 24 bits. [10110110]
One-time-pad The key has to be the same size (in bits) as the input message. Hence this is not going to work with the image (too many to enter) [10111111000010001111111101110010]
Block-cipher-chaining (Experimental) The message has to be evenly divisible by the key, e.g. if the message was 8 bits, the key would need to be 2, 4, or 8 bits so it leaves no remainder when divided by. Remember, each character in the input message is 1 byte and so 8 bits. Therefore 'Text' has 4 bytes or 24 bits. [10110110]
